Chosen Microinteraction: Pull to Refresh

Found on nearly every mobile app with a scrollable feed (e.g., Twitter/X, Instagram, Mail). The user drags the content downward to trigger a content reload.

1
Trigger
User-initiated trigger: The user places their finger (or cursor) on the content list and drags it downward past a threshold distance (~60 px). This is a manual, gesture-based trigger with no button involved. Try it in the demo above -- notice how nothing happens until you pull far enough.

System trigger: Some apps also auto-refresh when the view appears (e.g., switching tabs), but the canonical pull-to-refresh is a direct-manipulation gesture that puts the user in control.
Try it above -- drag down inside the feed!
2
Rules
What happens once triggered:
• The user must pull past a minimum threshold before the refresh activates; partial pulls snap back without refreshing.
• Once the threshold is crossed, the app sends a network request for the latest content.
• The spinner stays visible until the response is received or a timeout occurs.
• On success, new items are inserted at the top of the list. On failure, an error banner appears.
Try it above -- pull only halfway and release to see the cancel!
3
Feedback
Visual: A circular spinner appears proportionally as you pull. It rotates while loading, then morphs into a green checkmark when content arrives. Watch the spinner in the demo above!

Haptic: On iOS, a subtle haptic tap fires the moment the pull crosses the activation threshold. In the demo, a visual "bounce" simulates this.

Content shift: New items animate in from the top with a blue highlight, pushing existing content down smoothly.
Try it above -- watch the spinner transform into a checkmark!
4
Loops & Modes
Loop: The interaction is infinitely repeatable -- pull to refresh as many times as you wish. There is no cooldown, though rapid successive pulls typically return the same content if nothing has changed. Try refreshing the demo multiple times!

Mode change: If the device is offline, the interaction switches modes: the spinner may stop early and a "No connection" banner replaces new content. The same gesture now leads to a different feedback path.
Try it above -- refresh multiple times to see new posts appear!

Why Pull-to-Refresh Is a Great Microinteraction

Pull-to-refresh, originally designed by Loren Brichter for Tweetie (2008), is one of the most successful microinteractions in mobile design history. It works because the trigger is intuitive (the gesture mirrors physically pulling a list), the feedback is immediate and multi-sensory (visual spinner + haptic tap), the rules are simple and forgiving (partial pulls cancel gracefully), and the loop is natural -- users can repeat the action whenever they want with zero friction. It gives users a sense of direct control over content freshness, turning a mundane data-fetch into a satisfying moment of interaction.
